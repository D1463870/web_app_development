# 個人記帳簿系統產品需求文件 (PRD)

## 1. 專案概述
*   **背景與動機**：現代人生活忙碌，常常不清楚自己的錢花到哪裡去，導致無法進行有效的儲蓄或理財規劃。許多市面上的記帳軟體功能過於複雜，讓人難以養成持續記帳的習慣。因此，我們需要一個簡單、直覺的記帳工具。
*   **目標用戶**：大學生、新鮮人、小資族，以及所有想要開始理財但覺得現有工具太複雜的個人用戶。
*   **核心價值主張**：提供極簡且快速的記帳體驗，並透過清晰的統計圖表呈現收支狀況，幫助用戶輕鬆掌握自身財務健康度。

## 2. 功能需求

以下列出本系統的主要功能，並以使用者故事 (User Story) 描述：

1.  **收支紀錄管理 (CUD)**
    *   作為 **一般用戶**，我希望 **能快速新增、編輯與刪除一筆收支紀錄（包含金額、日期、類別、備註）**，以便 **準確記錄我每天的財務變動**。
2.  **收支明細檢視與篩選 (R)**
    *   作為 **一般用戶**，我希望 **能按月份或類別瀏覽收支明細清單**，以便 **回顧並對帳特定時期的花費紀錄**。
3.  **收支統計圖表**
    *   作為 **一般用戶**，我希望 **能看到自動生成的圓餅圖或長條圖**，以便 **一眼看出哪個類型的花費佔比最高，幫助我找出節流的方向**。
4.  **記帳類別自訂**
    *   作為 **一般用戶**，我希望 **能自訂或修改記帳的分類（如：伙食、交通、娛樂）**，以便 **讓記帳系統符合我個人的生活習慣**。
5.  **預算設定與提醒**
    *   作為 **一般用戶**，我希望 **能設定每個月的預算總額，並在接近預算上限時顯示視覺提醒**，以便 **控制開銷，防止超支**。
6.  **使用者帳戶管理**
    *   作為 **一般用戶**，我希望 **能註冊專屬帳號並登入系統**，以便 **保護我的財務隱私資料**。

## 3. 非功能需求
*   **技術限制**：
    *   後端框架使用 Flask (Python)。
    *   前端網頁渲染使用 Jinja2 模板引擎與原生 HTML/CSS/JS。
    *   資料庫使用 SQLite，方便輕量佈署與本地端測試。
*   **效能與安全考量**：
    *   操作回應時間（新增/讀取資料）應低於 1 秒，確保流暢的記帳體驗。
    *   密碼必須進行加密處理（Hash）後方可存入資料庫。
    *   需對使用者輸入進行驗證，以防止 XSS 及 SQL Injection 攻擊。

## 4. MVP 範圍

為了快速驗證市場需求，我們將功能分為不同階段：

*   **Must Have (核心必備)**
    *   使用者註冊與登入功能
    *   基礎收支紀錄（新增、檢視清單）
    *   系統預設的基本收支類別
*   **Should Have (重要但不急)**
    *   編輯與刪除現有收支紀錄
    *   按月份篩選清單
    *   簡單的圖表統計（如該月總收支、類別圓餅圖）
*   **Nice to Have (加分項目)**
    *   自訂收支類別
    *   預算設定與超支提醒
    *   CSV 資料匯出功能
